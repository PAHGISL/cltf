# CLTF reference outputs

These outputs are generated by the R `cltf` implementation and are the initial
numerical reference for the Python implementation.

## Cases

- `nsw_griffith_heavy_imazapic/` is the primary showcase case.
- `sa_minnipa_heavy_imazapic/` is the secondary regression case.

Both cases target layer-average resident concentration in provisionally
interpreted `ug/kg dry soil`, following the sampled-data structure. NSW bulk
density is from a credentialed SLGA v2 whole-earth point retrieval. SA bulk
density remains an explicitly labelled provisional SLGA v2-shaped offline
fixture until it is replaced by a credentialed point retrieval.

## Calibration note

The current CLTF equations depend on `mu * R_top` and `mu * R_bottom`.
Therefore exact equality of individual calibration parameters is not required
when comparing R and Python fits. Comparisons should focus on deterministic
forward outputs, objective values, mass balance, and the fitted transport
scales unless an external constraint resolves the scaling ridge.

`reference/tolerances.json` defines the numeric tolerances for cross-language
comparisons.
